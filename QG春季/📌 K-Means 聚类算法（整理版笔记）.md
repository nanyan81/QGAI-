# 📌 K-Means 聚类算法（整理版笔记）

## 一、基本概念

K-Means 是一种基于**欧式距离**的聚类算法，其核心思想是：

> **两个目标距离越近，相似度越大**

------

## 二、算法思想（牧师-村民模型）

### 2.1 模型描述

用一个形象例子理解：

- 有 **K 个牧师**（聚类中心）
- 有很多 **村民**（数据点）

过程如下：

1. 一开始，牧师随机站在某些位置（随机初始化 K 个中心）
2. 每个村民选择离自己**最近的牧师**（分配类别）
3. 牧师根据自己“负责的村民”，移动到这些村民的**中心位置**
4. 重复上述过程：
   - 村民重新选择最近的牧师
   - 牧师重新移动位置

👉 最终：

- 牧师位置不再变化 → 算法收敛

------

## 三、算法步骤（核心）

### Step 1：初始化

选择 K 个初始样本作为聚类中心：

```
a = a1, a2, a3, ..., ak
```

------

### Step 2：样本分配

对于每个样本 ( x_i )：

- 计算其到各个聚类中心的距离
- 分配到**最近的聚类中心对应的类**

------

### Step 3：更新聚类中心

对于每个类别 ( C_i )：

计算新的聚类中心：

aj=1∣Cj∣∑x∈Cjxa_j = \frac{1}{|C_j|} \sum_{x \in C_j} x

其中：

- ( C_j )：第 j 个簇
- ( |C_j| )：该簇样本数量
- ( x )：簇内样本

------

### Step 4：迭代

重复 Step 2 和 Step 3，直到满足停止条件：

- 聚类中心不再变化
- 或达到最大迭代次数

------

## 四、符号说明

- ( C )：算法过程中产生的中间变量（簇集合）
- ( C_i )：第 i 个类别
- ( |C_i| )：第 i 类中的样本数量
- ( \sum x )：表示该类中所有样本向量求和（例如所有“村民坐标”相加）

------

## 五、伪代码（你笔记里的代码整理版）

```text
获取 n 个 m 维数据（样本点）
随机生成 K 个 m 维点（初始中心）

while (未收敛)：
    for i in range(n):
        for j in range(k):
            计算样本 i 到中心 j 的距离
        将样本 i 分配到最近的中心

    for j in range(k):
        找出属于第 j 类的所有数据点
        将中心更新为这些点的均值
```

## 六、时间与空间复杂度

### 1️⃣ 时间复杂度

```
O(t × k × n × m)
```

其中：

- ( t )：迭代次数
- ( k )：簇的数量
- ( n )：样本数
- ( m )：特征维度

👉 特点：

- **运行速度较快（但数据大时仍有压力）**

------

### 2️⃣ 空间复杂度

```
O(m × (n + k))
```

👉 原因：

- 需要存储：
  - 所有样本（n × m）
  - 聚类中心（k × m）

------

## 七、优缺点

### 7.1 优点

1. **易理解**
2. **实现简单**
3. **计算复杂度低**

------

### 7.2 缺点

1. **对异常值（离群点）敏感**
2. **不适合多分类（复杂结构数据）**
3. **计算成本较高（大规模数据时）**
4. **容易陷入局部最优**

------

## 八、算法优化方向

1. **数据预处理**
   - 标准化 / 归一化
2. **合理选择 K 值**
   - 如：肘部法则（Elbow Method）
3. **高维映射**
   - 降维（PCA 等）
4. 其他改进：
   - K-Means++
   - Mini-Batch K-Means

------

## 九、一句话总结

> **K-Means = 不断“分组 + 更新中心”，直到稳定的聚类算法**
